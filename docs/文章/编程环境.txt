我2025年用的。

用了1个月，差点没破产。
$400多美金！问题是项目才开发了一大半，还没到测试阶段，搞得我心里没底，最恶心的是订阅后自动扣款。

<a id="故事2"></a>第二个故事：
Cursor太贵，那就换。全网说Trae+Qwen3-Code是国内最好的编程模型。
这不就是为我量身定做的吗。
中文、能自由切模型、简单任务用免费模型、复杂需求用Qwen3。
大厂产品，叠满了buff。我对字节简直是顶礼膜拜。
结果也是出人意料的好
1. 无法安装第三方Skills（如：Frontend，superpowers）。
2. 加功能像叠垃圾。每追加一个新功能，它给你加一堆代码。反正是实现了——你开始没发现，后来发现文件越来越多、代码越来越长。
3. 最大噩梦——测试修复。这是最要命的。你告诉它"修复扣除使用点数后应该刷新用户缓存的状态"。
   它先出一个流程方案，把你没想到的也考虑到了。我一看：这次稳了。流程很清晰啊。
   [图：Trae输出的"看起来很完美"的修复方案]
   结果呢——它排查很多文件，然后先把你需要的功能给实现了。至于以前已经通过的？它不管。

于是：
- 修完A的bug → 测试 → B出bug
- 修完B → 测试 → C出bug
- 修完C → 测试 → A又出bug

我看着我的应用是哭笑不得：
- 你说不能用——该有的功能都有了
- 你说能用——每个功能都有大大小小的问题

Token消耗比我家电表跑得还快。我的头发是大把大把地掉。

<a id="故事4"></a>第四个故事：

[图：]

账单出来那一刻，我盯着两个数看了半天：**$400 vs ¥18.79。**

代码一次跑通。核心功能全部过关，只有10%的细节动了第二下。
90%不是说100个bug自动修好90个。是以前提10个需求，6到7个要返工——修A坏B，改B崩C。现在10个里面最多1个要回头改。一次过。

差了快6倍的Token，12倍的钱。怎么做到的？

就5个东西。叠一起——不是加法是乘法。我用人话翻译一遍：


这五个工具组合在一起，可以理解为一套从**开发环境、AI大脑、工作流程到长期记忆**的完整编程助手方案。我用一个比喻帮你把它们串起来：

1. VS Code：”工作台”
这是微软开发的一款免费代码编辑器，你在这个台子上写代码、改文件、看结果。
它本身不写代码，但为所有其他工具提供了运行的底座。

2. Claude Code：”外国程序员”
把它装进VS Code后，就等于给你的工作台配了一个”老程序员”。
你可以像跟人说话一样告诉它需求（比如”帮我写一个登录页面”），它就能帮你生成、修改甚至检查代码。

3. DeepSeek：”大脑”
Claude Code默认自带一个”大脑”（Claude模型）。太贵，对中国用户不友好，经常找不到人，你还没地方说理。
而DeepSeek是另一个同样很聪明的AI模型，虽然差点但性价比高，我测试了国内很多模型，deepseek v4 pro很好用。

4. Superpowers：”项目经理”
AI虽然聪明，但有时会”自由发挥”——需求没问清楚就开始写代码，或为了改一个小功能顺手改了十几个文件。
Superpowers就是来解决这个问题的。它像一个严格的”项目经理”，强制AI在动手前必须先做这四步：
*   **先聊透需求**（头脑风暴）
*   **再写详细计划**
*   **按计划执行**
*   **最后自我审查**
有了它，AI就不会乱来了。

5. Claude-Mem：AI的”工作日志本”
Claude Code有个缺点：每次重新打开，它就像”失忆”了一样，不记得之前聊过什么。
Claude-Mem就是给它配的一个”工作日志本”。
它会自动记录下每一次的讨论、决策和踩过的坑。
下次再打开，AI翻翻日志就能”想”起来，不用你从头再解释一遍。

---

这套组合的核心思路就是：**用最好的工具（VS Code）+ 聪明的AI大脑（Claude Code + DeepSeek）+ 严格的流程管理（Superpowers）+ 可靠的长期记忆（Claude-Mem）**，让编程这件事变得高效又省心。
聪明的小伙伴会发现一个问题，claude code 不在做程序员，他是组合的核心把所有的工具进行链接，并在合适的时候调度他们为你所用。

单用任何一个工具，你还是会在某个点上被卡住。五个叠一起，一个坑都漏不掉。

为什么只有claude code 可以，答案很简单，别的工具不支持。

opencode目前支持了，我还没有亲测
