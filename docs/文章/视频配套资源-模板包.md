## 模板①：需求描述公式

每次让AI写代码，按这个格式填。复制到你的笔记里，改【】里的内容：

```
我要做：【具体功能——别用"登录"这种模糊词】

使用场景：【谁、什么情况下用】

交互流程：
1. 用户先……
2. 然后……
3. 如果成功→……
4. 如果失败→……

技术约束：
- 框架：【 】
- 文件放在：【 】

限制：
- 已有功能不要动：【 】

开始前请你：
1. 复述我的需求，确认你理解了
2. 列出要改哪些文件、每个文件负责什么
3. 等我确认再写代码
```

## 模板②：标准文件结构

让AI写代码前，先让它按这个结构列出文件。确认了再动手：

```
src/
├── modules/
│   ├── auth/          ← 登录、注册、找回密码
│   ├── product/       ← 商品列表、详情、搜索
│   ├── order/         ← 下单、支付回调、退款
│   └── user/          ← 个人信息、会员中心
├── components/        ← 可复用的按钮、表单、弹窗
├── services/          ← 所有API请求封装在这里
├── utils/             ← 工具函数（格式化、脱敏等）
└── styles/            ← 全局颜色、字号、间距变量
```

## 模板④：三句验证指令

AI写完代码后，按顺序问这三句。贴在你编辑器旁边：

```
1. "这段代码有没有安全隐患？
   列出所有可能的风险点，按严重程度排序。
   如果有，直接帮我修复。"

2. "有没有性能问题？
   比如重复请求、N+1查询、没做缓存的地方、
   内存泄漏风险？"

3. "帮我写3个测试用例，覆盖正常流程和异常流程。
   告诉我每一步怎么操作、预期结果是什么。
   我自己跑一遍验证。"
```
