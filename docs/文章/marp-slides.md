# 5个问题

沐言子宸

<!-- _class: lead -->

## 为什么你的AI代码
## 总是差一点？

**不是AI不会写。是它写的不是你想要的。**

---

## 五个阶段，一个死循环

1. **需求说不清** — AI不知道你在想什么，只能猜
2. **任务拆不开** — 一次塞太多，前面记得后面忘
3. **结构没设计** — 所有功能堆一个文件，改不动
4. **修改没规范** — 改A崩B，修B坏C
5. **结果没验证** — 能跑≠没问题，漏洞全在暗处

> 一路崩到底。把每一环补上，AI从实习生变助手。

## 问题一
# 需求说偏了

---

## 你说的 vs AI理解的

| 你说的 | AI理解的 |
|--------|----------|
| "帮我做一个登录" | 用户名 + 密码 |

| 你脑子里想的 | AI不知道的 |
|--------------|-----------|
| 手机号输入框 | 手机号 |
| 获取验证码按钮 | 短信验证 |
| 六位数字输入 | 验证码位数 |
| 成功跳首页 | 跳转逻辑 |

> **你没告诉它的，它只能猜。**

---

## 解法：需求描述公式

```
我要做：【具体功能】
使用场景：【谁、什么情况】
交互流程：
  1. 用户先……
  2. 如果成功→……
  3. 如果失败→……
技术约束：
  框架：【 】  文件放在：【 】
限制：
  已有功能不要动：【 】
---
开始前请你：
1. 复述需求，确认理解
2. 列出要改的文件
3. 等我确认再写代码
```

> **把AI当第一天入职的实习生，嚼碎了喂。**

## 问题二
# 一次塞太多

---

## AI的记忆像个杯子

```
"帮我做商城——注册、登录、商品列表、
详情、购物车、支付、会员、积分、订单、
退款、收藏、评价、优惠券……
哦对了，全部用绿色！"
```

写到第五个功能的时候——

**绿色？什么绿色？已经用上蓝色了。**

> 上下文窗口有限。东西扔太多，前面的被挤出去。

---

## 解法

# 每次只做一件事

做完 → 跑通 → 确认没问题 → 再开下一个

## 问题三
# 结构没设计

---

## 一个真实案例

**App.vue — 1700行**

登录、商品列表、个人中心、支付回调、订单管理

五个模块全在一个文件里。

改支付按钮颜色 → 翻600行 → 改了颜色 → 不小心删了订单查询函数。

**一个颜色，改了3个小时。**

---

## 解法：先画地图，再走路

```
src/
├── modules/
│   ├── auth/          ← 登录、注册
│   ├── product/       ← 商品列表、详情
│   ├── order/         ← 下单、支付
│   └── user/          ← 个人信息
├── components/        ← 可复用组件
├── services/          ← API请求
└── utils/             ← 工具函数
```

**让AI先列结构，你确认了再写代码。**

## 问题四
# 改A坏B

---

## 连锁崩溃

🔴 登录按钮改红色
        ↓
💥 支付按钮消失
        ↓
🔧 找回支付按钮
        ↓
💥 订单提示没了
        ↓
🔧 修复订单提示
        ↓
💥 **整个页面样式全乱**

**AI只盯着你指的一个点，不知道全局规则。**

---

## 解法：规矩文件

```markdown
# 项目规范 — AI修改代码前必读

## 修改铁律
- 不要删除任何已有功能
- 修改前先列出受影响的所有文件
- 每次只改一个功能点
- 改完后列出测试步骤

## UI规范
- 主色调：【 】  主按钮：【 】  危险按钮：【 】

## 禁止事项
- 不要用 any  不要硬编码  不要留 console.log
```

**每次让AI改代码前，先贴给它。**

## 问题五
# 能跑 ≠ 没问题

---

## 一个差点过审的页面

AI写了个表单，收集用户信息。

| 检查 | 结果 |
|------|------|
| 提交成功 | ✅ |
| 无报错 | ✅ |
| 页面正常 | ✅ |
| 接口鉴权 | ❌ 任何人知道URL就能调 |
| SQL参数化 | ❌ 可以被注入 |
| 参数校验 | ❌ 能拿到别人的数据 |

> **多问一句"安全吗？"，多花5分钟。**
> **隔壁团队没问——两周后数据被爬走。**

---

## 解法：写完代码必问三句

**1.** "有没有安全隐患？列出风险点，直接修复。"

**2.** "有没有性能问题？N+1查询、没做缓存、内存泄漏？"

**3.** "写3个测试用例，我跑一遍验证。"

> 让AI自己查一遍，你再验一遍。
