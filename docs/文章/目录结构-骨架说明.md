### 2.2 生成（用户运行skill后生成）

| 文件 | 生成时机 | 生成依据 |
|------|---------|---------|
| `docs/架构宪法.md` | 用户运行 `/架构宪法` | 用户填的项目类型/技术栈/规模 |

| 目录/文件 | 适用场景 | 备注 |
|----------|---------|------|

## 三、约束文件层次

```
CLAUDE.md（根级 — AI行为铁律，自动加载）
  │
  ├── src/api/CLAUDE.md（后端框架护栏 — 写API代码前自动关联）
  │
  ├── src/web/CLAUDE.md（前端框架护栏 — 写前端代码前自动关联）
  │
  ├── docs/架构宪法.md（系统边界 — 首次理解项目/改结构时手动调）
  │
  └── docs/项目宪法.md（代码规范 — 写具体代码时按层查）
```

**优先级：** CLAUDE.md > src/api/CLAUDE.md > 架构宪法 > 项目宪法

## 六、定制流程（用户视角）

```
1. git clone 或 Use this template → 得到骨架
2. 在 Claude Code 中打开项目
3. AI 自动读取 CLAUDE.md → 看到"首次使用"节
4. AI 引导问答：
   Q1: 项目类型？ → 后端 / 全栈 / CLI
   Q2: 后端语言/框架？ → Python+FastAPI / Go+Gin / Node+Express
   Q3: 数据库？ → MySQL / PostgreSQL / SQLite / 无
   Q4: 前端框架？ → 无 / Vue / React / 原生
   Q5: 安全等级？ → 一般 / 金融 / 医疗
5. AI 根据回答：
   - 修改 CLAUDE.md 中技术栈部分
   - 生成 docs/架构宪法.md
   - 生成 src/api/CLAUDE.md
   - 生成 src/web/CLAUDE.md（如选前端）
6. 用户开始开发
```

## 七、骨架维护规则

1. **不进业务代码。** 任何具体业务功能都不应进入骨架
2. **示例代码保持可运行。** main.py 必须能 `uvicorn main:app` 启动
3. **文件上限。** 骨架文件总数 ≤ 30 个，每个文件 ≤ 200 行
4. **skills指向最新版。** `.claude/skills/` 下的skill同步更新
