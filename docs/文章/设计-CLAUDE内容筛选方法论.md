## 一、三层筛选框架（通用）

CLAUDE.md 不是技术文档，是 **AI 行为护栏**。只放两类内容：

| 层 | 分类 | 判断标准 | 放不放 |
|----|------|---------|-------|
| 🚫 **必须放** | AI 不做就出**生产事故**的 | 并发竞态 / 资源泄漏 / 安全漏洞 / 架构越层 / 响应格式断裂 | 不放 → 项目出故障 |
| ✅ **可以放** | AI 不遵守代码就**烂**的 | 命名乱 / 异常滥用 / 配置不统一 / 日志污染 | 不放 → 代码难维护 |
| ❌ **不能放** | 放了**稀释约束效力**的 | 业务逻辑 / 通用编程知识 / 框架文档内容 / 具体表结构 | 放 → DeepSeek 分不清主次 |

### 筛子细则

#### 🚫 必须放（铁律级）

特征：缺了这条，用户直接损失钱/数据/安全。

- 架构越层：services 写 SQL、routes 直接操作数据库
- 并发竞态：重复下单、连接池耗尽、幂等缺失
- 资源泄漏：hot-reload 泄漏连接/Task/缓存
- 安全不可协商：SQL 参数化、身份来源、密钥管理
- 响应格式断裂：同一项目出现两种以上 JSON 返回格式

#### ✅ 可以放（质量级）

特征：代码能用，但修 bug 时想骂人。

- 命名规范：文件跨层对齐、函数动词+名词
- 异常使用规则：Service 层禁止 raise HTTPException
- 代码规范：类型注解、异步边界可见、import 顺序
- 配置管理：load_dotenv 只调一次、配置一致性
- 日志规范：级别选择、敏感脱敏

#### ❌ 不能放（噪音级）

特征：删了 ai 行为不变差。

- 业务逻辑：支付流程、文件上传逻辑、具体业务判断
- 通用编程知识：PEP8 缩进、变量不用拼音、函数不能太长
- 框架文档内容：Pydantic 怎么定义字段、FastAPI 路由装饰器用法
- 具体表/字段：user 表有哪些列、状态机枚举值
- "灵活性/扩展性"导向：工厂模式、策略模式的使用建议
- 与 ai 编码行为无关的：团队规范、代码 review 流程、git 分支策略

## 三、审查清单（排查用）

审一个 CLAUDE.md 是否合格，逐条过：

### 必查项

- [ ] 有没有架构越层禁止？（routes 不能写 SQL / services 不能手管连接）
- [ ] 有没有并发竞态处理？（防重 / 幂等 / 连接释放）
- [ ] 有没有资源清理机制？（池/Task/缓存）
- [ ] 有没有安全不可协商项？（SQL 参数化 / 身份来源 / 密钥管理）
- [ ] 响应格式有没有统一约束？
- [ ] 删掉最长的 3 条，AI 行为会不会变差？→ 如果不会，那是噪音

### 红线项（触及直接打回）

- [ ] 包含业务逻辑或具体业务字段？
- [ ] 包含框架教程级内容（FastAPI/Pydantic 怎么用）？
- [ ] 一处超过 250 行？（超过此数 DeepSeek 开始掉约束）

## 四、维护原则

1. **方法论不变，内容可调整** — 三层筛子是稳定的，具体约束项随项目技术栈微调
2. **审查时倒扣不叠加** — 如果某节同时满足"必须放"和"不能放"两个条件，以"不能放"为准
3. **行数是结果不是目标** — 先按筛子分类，再量化。不要为了凑行数塞内容
