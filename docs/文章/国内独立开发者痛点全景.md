### 社区总结的省钱方案

| 技巧 | 说明 |
|------|------|
| 预处理：用便宜模型"洗"需求 | 先在GPT-4o mini/DeepSeek/Gemini Flash提炼成结构化Prompt，再投喂给Claude/Cursor |
| 批处理：一次性问完所有问题 | 要求AI一次性列出所有澄清问题，你一次性回答(Q1:A, Q2:B, Q3:C) |
| 分治法：骨架先行，血肉后填 | 先做基础设施(DDL/目录结构/配置)，确认无误后再做业务逻辑 |
| 缩小上下文，强制模块拆分 | 咒语: `please ensure good module separation and design`；修bug时开新对话，只给出问题文件 |
| 善用缓存 | Claude prompt cache让重复输入打折计费，实测90%命中时成本大幅下降 |
| 模型降级策略 | 简单补全用Codeium/Tabnine(免费)；一般开发用Gemini 2.5 Pro/DeepSeek；复杂疑难用Claude Sonnet |
| 成本守卫+次数边界 | 连续失败3次→停止流水线→人工介入；设成本阈值(如$20)，超限暂停并确认 |
| Context Deduplication | 存储项目上下文一次，每次引用仅~100 Token，节省80-90% | [GitHub AI Context Stack](https://github.com/aandersen2323/ai-context-stack) |

### 社区共识的解决思路

1. 把自己当"项目经理"：拆解需求成小模块，只让AI完成小模块，自己组装掌控全局
2. 规格驱动开发(SDD)：用OpenSpec+Superpowers组合，先明确标准再规范编码
3. 先自己写再让AI辅助：先熟悉语言框架，保证有能力改AI的代码
4. Review不可省略：每次AI生成后必须review，不要盲目keep all
5. 测试先行(TDD)：用测试用例定义期望行为，规避AI的脑补和遗忘
6. 从不理解就不合并：这是不可妥协的底线
7. 每次生成后追问安全风险："这段代码有哪些潜在的安全漏洞？"

### 心理层面的三重困境

**第一道坎：从"手艺人"到"生意人"的身份撕裂。** 本质上是优秀的手艺人，创业后80%时间在做报价/催款/营销/客服，感到身份撕裂。

**第二道坎：决策疲劳。** 每个决策无人讨论、无人审核、无人兜底，长期独自决策→决策质量下降→焦虑加剧→倦怠提前。

**第三道坎："木桶短板"效应。** 一人公司的生死取决于你最弱的那块板。产品再好不会卖=库存，创意再牛不懂法务=被一个条款击倒。

> "一人公司最难的不是钱或能力，而是在全世界最孤独的决策环境里完成和自己人性的惨烈内战。" — [知乎](https://www.zhihu.com/question/1929952636016268030/answer/2043704054703031110)

### 头部创业者的务实建议

1. **先验证获客能力，再投入产品开发** — 先回答：目标用户在哪？通过什么渠道触达？谁愿意付费？
2. **选择"小而痛"的垂直领域** — 大公司看不上但特定群体愿付费的小众需求
3. **设定明确止损线** — 3个月月收入未达$500/6个月未跑通付费转化，果断切换方向
4. **构建"反脆弱"收入结构** — 订阅+一次性销售+咨询/顾问，多产品并行
5. **承认能力边界** — 商业模式跑通后引入合伙人或外包；"一人公司"不是目的，是手段
