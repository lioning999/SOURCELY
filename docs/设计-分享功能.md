## 一、核心决策

### 为什么做

| 维度 | 判断 |
|------|------|
| **传播价值** | 使用场景自带传播——卖家查完品立刻想发群里问同行意见。决策卡图片 = 最好的广告素材 |
| **获客成本** | [推测] 30-40% 新用户来自老用户分享，比投 Facebook 广告便宜一个数量级 |
| **用户习惯** | 东南亚卖家群聊文化极重（Zalo/LINE/WhatsApp 群），截图分享是日常行为 |

### 做什么

| 决策 | 结论 | 依据 |
|------|------|------|
| **分享形式** | **图片导出（PNG）** | 不需要后端，零 Apify 成本，任何 App 都能发 |
| **系统分享** | 同时发 **图片 + 链接** | `navigator.share()` 支持，一手转化率最高（~30%） |
| **链接类型** | V1 链接到 **首页** `sourcely.com` | 不做产品快照页（排 V1.1）。群友点进来 → 看到搜索框 → 自然会试 |
| **工期** | +1 天 | 纯前端，html2canvas + 隐藏 DOM 渲染 |
| **Apify 影响** | **零** | 图片是静态导出，链接指向首页（无产品查询） |

### 不做什么

| 不做 | 原因 | 排期 |
|------|------|------|
| 活链接分享（点开调 Apify） | Apify 陷阱——每个点开的人触发一次调用 | 不做 |
| 快照链接（产品详情页） | 需要后端支持，工期 +2 天 | V1.1 |
| QR 码 | 手机→手机场景无效（自己扫不了自己屏幕上的图） | 不做 |
| 分享激励（分享免服务费） | V1 先验证自然传播，不搞运营活动 | V2 |

## 二、按钮设计

### 布局：一主一辅两个按钮

| 位置 | 触发场景 | 覆盖占比 |
|------|---------|---------|
| **主按钮**：综合决策卡底部，信任条下方、Tab 上方 | 用户读完价格→判词→信任条，立刻想分享 | ~80% |
| **辅按钮**：拿样 Tab 底部，WhatsApp CTA 上方 | 用户翻完三个 Tab 深入了解后想分享 | ~20% |

### 样式

```
次要按钮风格——不抢 WhatsApp CTA 的视觉权重

圆角 20px，边框 1px solid var(--line)
背景 var(--paper-2)，文字 "📤 分享决策卡"
font-size: 13px，padding: 8px 18px
hover 边框变金色
```

**设计要点：**
- 图标明确（📤 或 SVG 分享图标），一眼知道是分享
- 文字明确（"分享决策卡"），告诉用户分享出去的是什么
- 和"查看 1688 原页面"拉开视觉距离——分享在上面（高频），去 1688 在下面（低频）

## 三、交互流程

```
用户点击 [📤 分享决策卡]
        │
        ▼
前端渲染隐藏的分享卡 DOM → html2canvas → PNG blob
        │
        ▼
底部弹出半屏 Sheet（预览）
  ┌─────────────────────────┐
  │                         │
  │     [分享图预览]        │
  │                         │
  ├─────────────────────────┤
  │  [💾 保存图片]          │  ← 下载 PNG 到相册
  │  [📱 分享到应用]        │  ← navigator.share({ files, url })
  └─────────────────────────┘
        │
        ▼
用户确认
  └─ 保存图片 → 下载到手机相册
  └─ 分享到应用 → 系统分享面板 → WhatsApp/Zalo/LINE/...
      同时发送：图片 + sourcely.com 链接
```

### 为什么半屏预览是必须的

用户分享前需要确认——"这张图好看吗？内容对吗？"不预览直接下载，发出去发现图不对，回不了头。

## 四、分享图设计

### 设计原则

1. **不要截取页面 DOM。** 手机浏览器截图一致性差（字体缺失、iOS/Android 差异）
2. **用隐藏 DOM 专门渲染。** 750px 宽，CSS 全部内联，字体用系统栈
3. **信息缺口策略。** 只放综合决策卡内容，不放完整报告——让群友想来网站看更多
4. **品牌条要醒目。** 不是小水印，是深色背景条，群聊缩略图里也看得清

### 分享图内容

```
┌─────────────────────────────────┐  宽 750px
│                                 │
│         SOURCELY                │  品牌 logo（顶部，轻量）
│                                 │
│  ┌───────────────────────────┐  │
│  │                           │  │
│  │     [商品主图 正方形裁剪]  │  │  ← 取第一张 product image
│  │                           │  │     750×750，居中裁剪
│  └───────────────────────────┘  │
│                                 │
│  卡通可爱手编织小蒲扇子女夏天    │  ← 标题 ≤ 3 行，截断加…
│  新款便携随身手摇扇照道具扇子   │
│                                 │
│  ¥7.5 – ¥7.8 / 把              │  ← 价格，品牌色深红
│  MOQ 2件 · 已售 4,800+ 件      │
│                                 │
│  🏷 源头旗舰   免费拿样          │  ← 标签行
│                                 │
│  🕐 6年老店  ★ 99.3%好评       │  ← 信任条
│  ↻ 76.4%回头  🌍 90+跨境买家   │
│                                 │
│  ┌─────────────────────────┐   │
│  │  🔍 SOURCELY            │   │  ← 品牌条：深蓝背景 #1E3A5F
│  │  30秒看懂1688工厂        │   │     白色文字，高 48px
│  │  sourcely.com           │   │     字号 13px，在缩略图中可辨认
│  └─────────────────────────┘   │
└─────────────────────────────────┘
```

### 品牌条设计规范

| 属性 | 值 | 理由 |
|------|-----|------|
| 背景色 | `#1E3A5F`（`--brand-deep`） | 和上方内容明显分层 |
| 文字色 | `#FFFFFF` | 高对比度 |
| 品牌名 | `SOURCELY` 大写加粗 | 口口相传靠品牌名，不是 URL（雷军） |
| 副标题 | `30秒看懂1688工厂` | 一句话说清价值主张 |
| 域名 | `sourcely.com` | 足够短，不需要短链 |
| 高度 | ≥ 48px | 确保群聊缩略图里也看得清 |
| 字号 | ≥ 13px | 最小可读尺寸 |

### 信息缺口：分享图故意不留的内容

| 不留 | 理由 |
|------|------|
| 完整产品规格 | 群友想来网站看更多 → 触发访问 |
| 工厂详情 | 同上 |
| 拿样费用计算 | 同上 |
| 判词展开 | 只放综合判词一句话，不放三个 Tab 的详细判词 |

## 五、溯源方案

### 三重保障

```
一手传播（用户从 Sourcely 直接分享）：
  navigator.share({ files: [pngBlob], url: "https://sourcely.com" })
  → WhatsApp/Zalo 同时显示：图片 + 可点击链接
  → 预计转化率 ~30%

二手传播（群友保存图片再转发）：
  图片底部品牌条 → 群友手动搜索 "Sourcely" 或输入域名
  → 预计转化率 ~5-10%

口口相传：
  群友问"这是什么工具？"→ 分享者回答品牌名
  → 预计转化率 ~5%
```

### 为什么不用 QR 码

| 问题 | 说明 |
|------|------|
| **物理限制** | 手机上的图 → 需要另一台手机或镜子才能扫。自己扫不了自己屏幕 |
| **额外步骤** | 保存图片 → 打开相册 → 长按识别（仅 iOS 17+）→ 流程太长 |
| **占空间** | QR 码最小也需要 60×60px，在图片上占用宝贵视觉空间 |
| **结论** | 手机→手机分享场景，QR 码是无效手段 |

### 系统分享 API 行为

```javascript
// 同时分享图片 + 链接
const blob = await html2canvas.toBlob(shareCard);
const file = new File([blob], 'sourcely-decision-card.png', { type: 'image/png' });

await navigator.share({
  files: [file],
  title: 'Sourcely 产品决策卡',
  url: 'https://sourcely.com'
});
```

| 平台 | 行为 |
|------|------|
| **WhatsApp** | 图片 + 链接同时发送，链接可点击 |
| **Zalo** | 图片 + 文本（含 URL），URL 可点击但可能不展开预览 |
| **LINE** | 图片 + 文本（含 URL） |
| **Facebook Messenger** | 图片 + 链接预览卡片 |

## 六、技术实现

### 技术选型

| 方案 | 依赖 | 说明 |
|------|------|------|
| **html2canvas** | 前端库 ~30KB | 渲染隐藏 DOM → Canvas → PNG。纯前端，零后端 |
| 替代：Playwright 截图 | 后端 Python | 需要部署浏览器内核，V1 杀鸡用牛刀 |

**选 html2canvas。** 理由：纯前端执行，秒级完成，不经过后端，零服务器负载。代码量 ~80 行 JS。

### 实现要点

1. **分享图用独立隐藏 DOM。** 不截取页面可见区域——跨平台渲染一致性差
2. **CSS 全部内联。** 分享图 DOM 的样式写在 `style` 属性里，不依赖页面样式表
3. **字体用系统栈。** `-apple-system, BlinkMacSystemFont, 'Segoe UI'`，不依赖 Web Font
4. **图片 URL 先转 base64。** 避免 html2canvas 跨域图片问题
5. **PNG 压缩。** 目标 200-300KB（越南卖家：4G 网络传大图很慢）
6. **渲染完成后销毁隐藏 DOM。** 避免内存泄漏

### 伪代码

```javascript
function buildShareCard(productData) {
  const card = document.createElement('div');
  card.style.cssText = 'position:fixed;left:-9999px;width:750px;...';
  card.innerHTML = `
    <div style="padding:24px;...">
      <!-- 品牌 logo -->
      <div style="font-size:18px;font-weight:700;...">SOURCELY</div>
      <!-- 主图 -->
      <img src="${productData.mainImage}" style="width:100%;aspect-ratio:1;object-fit:cover;...">
      <!-- 标题 -->
      <div style="font-size:20px;font-weight:700;...">${truncate(productData.title, 3)}</div>
      <!-- 价格 -->
      <div style="font-size:28px;color:#D6432F;...">¥${productData.priceMin} – ¥${productData.priceMax}</div>
      <!-- 标签 + 信任条 + 品牌条 -->
      ...
    </div>`;
  document.body.appendChild(card);
  return card;
}

async function shareCard(productData) {
  const card = buildShareCard(productData);
  const canvas = await html2canvas(card, { scale: 2 });
  card.remove();  // 立即销毁

  const blob = await new Promise(r => canvas.toBlob(r, 'image/png', 0.85));
  const file = new File([blob], 'sourcely-decision.png', { type: 'image/png' });

  // 预览 Sheet
  showPreviewSheet(URL.createObjectURL(blob), file);
}

function showPreviewSheet(imageUrl, file) {
  // 底部半屏弹窗：预览图 + [保存图片] + [分享到应用]
  // [分享到应用] → navigator.share({ files: [file], url: 'https://sourcely.com' })
  // [保存图片] → <a download> 触发下载
}
```

## 七、明确不做（V1 范围外）

| 不做 | 原因 | 排期 |
|------|------|------|
| 链接分享到产品详情页 | 快照页需要后端支持 + 2 天工期 | V1.1 |
| QR 码 | 手机→手机分享物理上扫不了 | 不做 |
| 分享图可编辑（标题裁切等） | 过度设计，V1 固定模板即可 | 不做 |
| 分享统计/追踪 | V1 用户量不需要 | V1.1 |
| 分享激励（分享 N 次免服务费） | 先验证自然传播 | V2 |
| 图片加 URL 短链 | `sourcely.com` 已经够短 | 不需要 |
