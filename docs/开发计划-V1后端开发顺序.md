## 2. Phase 1A — 项目骨架（P0）

**目标：`uvicorn main:app` 能启动，连接池可用，异常体系就位。**

### 文件清单（4 个）

```
src/api/
├── requirements.txt          ← 依赖清单
├── utils/exceptions.py       ← AppError 异常体系
├── utils/jwt.py              ← JWT 签发 + 解码
├── database.py               ← MySQL 连接池（aiomysql）
└── main.py                   ← FastAPI 入口 + CORS + lifespan
```

### 执行顺序

| 步骤 | 文件 | 做什么 |
|------|------|--------|
| ① | `requirements.txt` | 列出依赖：fastapi, uvicorn, aiomysql, pyjwt, google-auth, httpx, python-dotenv |
| ② | `utils/exceptions.py` | AppError 基类 + ExternalServiceError + ResourceNotFoundError + DatabaseError |
| ③ | `utils/jwt.py` | `create_token(user_id, email)` + `decode_token(token)` |
| ④ | `database.py` | aiomysql 连接池，`autocommit=False`，提供 `get_connection()` |
| ⑤ | `main.py` | 创建 FastAPI app，注册 CORS + lifespan（init/destroy pool） |

### 验证标准

```bash
pip install -r requirements.txt
uvicorn main:app --reload
# → INFO: Uvicorn running on http://127.0.0.1:8008
# → /docs 能打开 Swagger
```

## 3. Phase 1B — Google 登录（P1）

**目标：前端发 Google credential，后端返回 JWT access_token。**

### 文件清单（5 个后端 + 1 个前端）

```
src/api/
├── middleware.py              ← JWT 验证中间件
├── adapters/google_auth.py   ← Google id_token 离线验证
├── repositories/user_repo.py ← users 表 CRUD
├── services/auth_svc.py      ← 登录/注册业务编排
└── routes/auth.py            ← POST /api/auth/google

src/web/
└── static/js/components.js   ← 加入 Google 登录按钮逻辑
```

### 执行顺序

| 步骤 | 文件 | 做什么 |
|------|------|--------|
| ⑥ | `middleware.py` | 拦截请求 → 检查 `Authorization: Bearer <token>` → 解码 JWT → `request.state.user_id` |
| ⑦ | `adapters/google_auth.py` | 调 `google-auth` 库验证 id_token，检查 aud/exp/iss |
| ⑧ | `repositories/user_repo.py` | `get_by_google_id()` / `create()` / `update_last_login()` |
| ⑨ | `services/auth_svc.py` | 编排：验证 credential → upsert user → 签发 JWT |
| ⑩ | `routes/auth.py` | `POST /api/auth/google` 端点，Pydantic 校验入参 |
| ⑪ | `components.js` | 加入 `initGoogleLogin()` / `handleCredential()` / `checkAuth()` / `logout()` |

### 验证标准

```
curl -X POST http://127.0.0.1:8008/api/auth/google \
  -H "Content-Type: application/json" \
  -d '{"credential":"<Google id_token>"}'

# → {"access_token": "eyJ...", "user": {"id":1, "name":"...", "email":"...", "avatar_url":"..."}}
```

### 关键设计决策

| 决策 | 结论 | 理由 |
|------|------|------|
| JWT 存储位置 | sessionStorage | 关标签即清，比 localStorage 安全 |
| 登录成功刷新 | 用 `location.reload()` | V1 简单，V1.1 改 DOM 局部更新 |
| 后端验证方式 | google-auth 离线验证 | 比调 tokeninfo API 更快 |
| 认证方式 | 只做 Google OAuth | 东南亚卖家都有 Gmail，最简 |
