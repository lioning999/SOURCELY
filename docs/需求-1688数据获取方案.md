## 需求背景

Sourcely 的核心功能：用户粘贴 1688 商品链接 → 页面展示产品分析（好评率、成交量、起批条件等）+ 工厂背调（认证、回头率、发货速度等）。

**当前状态：** Demo 写死静态数据（女士连体泳衣）。需要接入真实 1688 数据。

**目标用户：** 越南、泰国、印尼等东南亚 Shopee/Lazada 小卖家。用户无法注册 1688（需中国手机号+身份证+支付宝实名）。

## 技术选型

### 方案对比

| 方案 | 原理 | 成本 | 稳定性 | Demo 适用 |
|------|------|------|--------|:---:|
| 1688 官方 API | 入驻 1688 开放平台，调 `1688.item_get` | 免费 | ⭐⭐⭐⭐⭐ | ❌ 需企业资质（邀约制） |
| 自建 Playwright 爬虫 | 服务器跑 Chrome，模拟用户操作 | 免费 | ⭐⭐⭐ | ❌ 服务器无桌面 + 数据中心IP被封 |
| Apify 云爬虫 | 调 Apify 平台 API，平台负责代理+解析 | ~$4/千条 | ⭐⭐⭐⭐ | ✅ |

**选型结论：** Demo 阶段用 **Apify Zen Studio 1688 Wholesale Scraper**。验证模式后考虑自建。

### 选中的 Apify Actor

- **名称：** [1688 Wholesale Scraper](https://apify.com/zen-studio/1688-wholesale-scraper) (by Zen Studio)
- **用户量：** 230 MAU（Apify 平台最高）
- **数据字段：** 50+（价格阶梯、MOQ、销量、回头率、供应商评分、服务标签、跨境平台标签等）
- **价格：** $3.99/千条结果
- **速率：** 250+ 产品/分钟

## 实施计划

### 步骤 1：后端 `/api/analyze` 端点（4h）

**文件：**
- `src/api/adapters/apify_adapter.py` — Apify API 封装
- `src/api/services/analyze_service.py` — 业务编排 + 数据映射
- `src/api/routes/analyze.py` — 路由 + 参数校验

**流程：**

```
POST /api/analyze
Body: {"url": "https://detail.1688.com/offer/123456.html"}

1. routes/ 提取 URL 中的 offerId（正则 /\d{8,}/）
2. services/ 调 adapters/apify_adapter
3. adapters/ POST Apify API → 轮询 dataset → 返回原始 JSON
4. services/ 映射 Apify 字段 → 前端格式
5. routes/ 返回标准响应 {code: 200, data: {...}}
```

**Apify 调用逻辑：**

```python
# adapters/apify_adapter.py
# 1. POST https://api.apify.com/v2/acts/zen-studio~1688-wholesale-scraper/runs
#    Body: {"offerUrls": ["https://detail.1688.com/offer/{id}.html"]}
# 2. 拿到 runId → 轮询 dataset items（间隔 1s，最多 15s）
# 3. 返回 items[0] 或 None
```

### 步骤 2：数据映射 + 前端对接（3h）

**文件：**
- `src/api/services/analyze_service.py` — 映射函数
- `src/web/static/js/pages/index.js` — 前端调用

**前端改动：** 搜索按钮从 `setTimeout` 模拟改为 `fetch('/api/analyze', ...)` 真实请求。返回后填充 DOM 的 11 个字段。

### 步骤 3：缓存 + 容错（1h）

**缓存：** 内存 dict，key=offerId，避免重复调 Apify（省钱）。后续换 Redis。

**限流：** 每 IP 每日 3 次免费调用。超限返回 429 + WhatsApp 解锁提示。

**容错：**
- Apify 超时（15s）→ 504 + "分析超时，请重试"
- Apify 返回空 → 200 + "该链接暂无数据"
- Apify Token 失效 → 500 + 日志告警

## 数据映射表

| Demo 页面字段 | Apify 原始字段 | 转换 |
|--------------|---------------|------|
| 标题 | `productTitle` | 直接 |
| 主图 | `imageUrls[0]` | 直接 |
| 价格区间（USD） | `priceRangeDisplay` | CNY / 7.2 换算 |
| 7天无理由 | `serviceTags` | 包含"7天"→OK |
| 好评率 | `goods_score` | 转百分比 |
| 近30天成交 | `saledCount` | 直接 + ¥估算 |
| 起批条件 | `moq` + `priceTiers` | 直接 + 1688 原链接 |
| 1688认证商家 | `supplierVerificationFlags` | 任一为真→OK |
| 回头率 | `repurchaseRate` | 直接（百分数） |
| 发货速度 | `logistics_score` | ≥4.0→OK，<4.0→NO |
| 货描相符 | `goods_score` | 直接（x.x/5.0） |

**Apify 无法获取的字段（后续迭代）：**

| 字段 | 原因 | 解决方案 |
|------|------|---------|
| 开店时长 | 不在商品页，在店铺主页 | 二期：单独爬店铺页 |
| 经营模式（生产厂家/经销商） | 同上 | 同上 |
| 实地验厂 | 第三方认证数据 | 手动标注或接 SGS API |
| 跨境专供 | Apify 无此标签字段 | 需查 1688 跨境寻源通 API |

## 风险与容错

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| Apify 冷启动慢（5-10s） | 高 | 用户体感差 | 进度条 + 缓存热数据 |
| Apify 服务宕机 | 低 | 分析不可用 | 前端 fallback 提示 + WhatsApp 人工 |
| Apify 作者弃坑 | 低 | 需切其他 Actor | 备用 Actor：devcake/1688-com-products-scraper |
| 1688 页面改版 | 中 | Actor 解析失败 | Apify 社区会修，或自建 Playwright 兜底 |
| 日调用超 3 次 | 高 | 免费用户卡住 | 返回 WhatsApp 联系解锁 |
